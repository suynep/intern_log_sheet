### Technical "Today I Learned" (TIL)

- Layouts in Flutter, and how they can be embellished using various Widget properties.
- With the multitude of Material Widgets (that adhere to the [Material Design Specifications](https://m3.material.io/)) it's super easy to create neat-looking, cross-platform applications (see the [Tasks Completed](#tasks-completed) Section)
- Three Types of trees exist in A Flutter Application's Lifecycle: **Widget Tree** *(what to display)*, **Element Tree** *(where to display)*, **Render Tree** *(how to display, the low-level graphics API stuff)*. Us developers usually work with the **Widget Tree**, and the framework manages the **Element Tree** and the **Render Tree**.
- Flutter layouts are created by composing widgets like Row, Column, Container, and Expanded, allowing flexible arrangement and responsive design without traditional layout files.
- The widget tree architecture encourages modular design, making it easier to separate UI components into reusable parts.
